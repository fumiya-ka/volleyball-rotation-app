# バックログ（持ち越しタスク）

**作成日**: 2026-06-17
**位置づけ**: 未着手の機能拡張メモ。実装時はこの分類に沿って `buildSequence` / `sequenceConstants.json` / `PhaseTabs`（パターン選択UI）/ 解説テキストを拡張する。

## 1. ラリー局面に「チャンスボール」を追加

- 現状のラリー局面は **被スパイク**・**攻撃** の2つのみ。
- ここに **チャンスボール**（相手からの緩い返球を拾って攻撃につなぐ局面）を追加する。

### 分類: 返球先ポジション別 — 6パターン
チャンスボールが**自陣のどのポジションに返ってくるか**でパターン分けする。各ポジション(1〜6)に1パターン、計 **6パターン**。

| # | 返球先 | 備考 |
|---|--------|------|
| 1 | ポジション1（右後） | |
| 2 | ポジション2（右前） | |
| 3 | ポジション3（中前） | |
| 4 | ポジション4（左前） | |
| 5 | ポジション5（左後） | |
| 6 | ポジション6（中後） | |

**展開: 攻撃局面へ連結する**（決定）
- チャンスボールを受けた後はセッターへ繋ぎ、そのまま **攻撃局面（下記3）へ連結** させる（単体局面にしない）。
- 返球先ポジションに応じてレシーブ担当が変わり、その後はローテに応じた攻撃へ移行する。

---

## 2. 被スパイク（ディフェンス）のパターン分類 — 6パターン

相手スパイクの **コース3種 × 球質2種 = 6パターン** に分類する。

| コース＼球質 | 強打 | フェイント |
|---|---|---|
| レフト（自陣から見て左） | ① | ④ |
| センター | ② | ⑤ |
| ライト（自陣から見て右） | ③ | ⑥ |

- コース: レフト / センター / ライト（相手アタッカーの打点と自陣の受け手・ブロックが変わる）
- 球質: 強打 / フェイント（軌道・スピード・ディグ位置が変わる）

> 現状は「相手アウトサイド＝自陣ライトから強打→クロスを左後ろがディグ」の1パターンのみ実装。6パターン化に伴い、ブロック対象・ディグ担当・ボール軌道（強打＝速く低い／フェイント＝山なりで手前）を切り替える。

---

## 3. 攻撃のパターン分類 — 9種類

トスの種類 × コースで **9種類** に分類する。

| 種類 | コース | 数 |
|------|--------|----|
| オープントス | レフト / ライト | 2 |
| セミ | センター | 1 |
| 平行トス | レフト / ライト | 2 |
| クイック A / B / C / D | センター | 4 |
| **合計** | | **9** |

- オープン: レフト・ライト（高く幅のあるトス）
- セミ: センター（中速）
- 平行: レフト・ライト（速く低いトス）
- クイック A〜D: センター付近（A=近い〜D=遠い等、4種）

**現状の攻撃テンポ選択（1st/2nd/3rd）はこの9分類に置き換える**（決定／拡張ではなく置換）。トス先・スパイカー・助走・打点（手アライメント）をパターンごとに定義。

## 実装メモ（共通）

- **データ**: 各局面の「パターン」を `sequenceConstants.json` にパターン別パラメータとして追加し、`buildSequence(rotKey, phase, pattern, C)` のようにパターン引数で分岐。
- **UI**: `PhaseTabs` にパターン選択（攻撃テンポ選択の汎用化）を追加。局面ごとに選択肢数が変わる（被スパイク=6 / チャンス=6 / 攻撃=9）。
- **状態**: `sceneStore` にパターン選択 state を追加（フェーズ切替時のリセット対象）。
- **解説**: `descriptions` をパターン別に拡張。
- **既存方針の踏襲**: 役割ベース配置 / 打点＝手アライメント / ボール軌道は単純なサインカーブ / ボール保持0.1秒以内。
