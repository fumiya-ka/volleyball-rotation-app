## 1. システム概要

バレーボールの6ローテーション × 4フェーズ（レセプション・サーブ・ディフェンス・攻撃）における選手とボールの動きを3Dアニメーションで可視化・解説するWebアプリケーション。

---

## 2. 目的・背景

- バレーボールのローテーションルールや各フェーズの動きは、静止画や文章では直感的に理解しにくい
- 3Dアニメーションで視覚的に動きを再現し、選手・指導者・観戦者が戦術を理解しやすくする

---

## 3. 機能要件

### 3.1 コア機能

| 機能 | 概要 |
|------|------|
| フェーズ切替 | ① 相手サーブ（レセプション）/ ② 自陣サーブ / ③ 被スパイク（ディフェンス）/ ④ 攻撃 の4フェーズを切替 |
| ローテーション選択 | S1〜S6 の6ローテーションを切替。全フェーズで共通 |
| 攻撃テンポ選択 | 1st（クイック）/ 2nd（セミ）/ 3rd（高め）を切替。攻撃フェーズのみ表示 |
| 3Dアニメーション再生 | 選手6名+ボールをリアルタイム3Dアニメーションで描画 |
| 再生コントロール | 再生/一時停止、シークバー、再生速度（0.5x / 1x / 1.5x / 2x） |
| 解説テロップ | フェーズ × ローテーション × テンポ に応じた解説テキストを表示。要約表示＋「詳しく」展開トグル（展開時は背景を透過しコート視認性を確保） |
| カメラ操作 | マウスドラッグで3Dシーンを任意角度から閲覧可能（OrbitControls） |

### 3.2 開発専用機能（DEVモード）

| 機能 | 概要 |
|------|------|
| DEVパネル | 選手座標・ボール軌道・タイミングをスライダーでリアルタイム調整 |
| ドラッグ操作 | 選手・ボールをドラッグで手動配置（位置オーバーライド） |
| JSON出力 | 調整済みパラメータを `sequenceConstants.json` としてダウンロード |

## 5. システム構成

```
ブラウザ
  └── SPAアプリ（React + Vite）
        ├── UIレイヤー（HTML/CSS）
        │   ├── TopBar        — タイトル・ローテーション表示
        │   ├── PhaseTabs     — フェーズ/ローテーション/テンポ選択
        │   ├── Telop         — 解説テロップ
        │   └── PlaybackControls — 再生操作
        └── 3Dレイヤー（Three.js / React Three Fiber）
            ├── Scene         — Canvas・カメラ・ライティング・ポストエフェクト
            ├── Court / Net   — コート・ネット描画
            ├── Player / Ball — 選手・ボールメッシュ
            └── SequenceAnimator — アニメーションループ
```

デプロイ先は Vercel の静的ホスティング。サーバーサイド処理はなく、すべてクライアントサイドで完結する。

## 8. データ設計（概要）

### 8.1 選手定義

6名固定：S（セッター）、OP（オポジット）、OH1・OH2（アウトサイドヒッター×2）、MB1・MB2（ミドルブロッカー×2）。

### 8.2 ローテーション定義

6ローテーション（S1〜S6）× ポジション番号（1〜6）の対応表。ポジション番号はコート座標にマッピング。

### 8.3 アニメーションデータ

フェーズ × ローテーションごとにキーフレーム配列を動的生成。タイミング・座標パラメータは `sequenceConstants.json` で一元管理。

### 8.4 状態管理

Zustand の単一ストアで再生状態・UI選択・3D座標を一元管理。フェーズ/ローテーション切替時に時刻・位置オーバーライドをリセット。
